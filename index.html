QA Project Portfolio

TEST SUITE 1: Urban Scooter (Full Stack Testing)

Category: Manual Testing / API / Mobile

1. The Challenge

As part of my comprehensive professional practice, Urban Scooter required validation of the complete scooter rental and tracking flow. The main challenge was identifying real defects and undocumented behaviors, verifying critical synchronization between the Web, Mobile App, and API. The goal was to ensure a stable and logical user experience.

2. My Role and Tools

  • Role: QA Tester (E2E Execution, Documentation).
  • Tools: JIRA, Postman, Chrome DevTools, Charles Proxy, TestRail.

3. The Process

I approached the project by decomposing requirements using mind maps to visualize expected behavior.

  • Test Design: I applied equivalence partitioning and boundary value analysis to optimize coverage.
  • Cross-Platform Execution: I validated the rental form on Web, tested offline states and notifications on Mobile, and executed API tests to confirm HTTP state transitions.
  • Documentation: I recorded every finding in JIRA with clear evidence for developers.

4. Results and Conclusions

This project revealed critical defects, including order duplication and hidden dependencies between the front and back-end. I delivered solid documentation demonstrating real inconsistencies, validating my ability to manage a full QA cycle with critical thinking.

TEST SUITE 2: Urban Grocers – Web UI Automation

Category: Automation / Python / Selenium

1. The Challenge

Automatic validation was required for the business rules of the "Name" field during product Kit creation. Manual validation was fragile and prone to false negatives due to dynamic DOM elements and timing-dependent behaviors.

2. My Role and Tools

  • Role: QA Automation Engineer.
  • Tools: Python, Pytest, Selenium WebDriver, Git/GitHub, VS Code.

3. The Process

To solve this, I developed a robust automation framework:

  • Architecture: I implemented the Page Object Model (POM) pattern to make the code scalable and easy to maintain.
  • Coverage: I automated positive, negative, boundary value scenarios, and special character handling.
  • Stability: I used WebDriverWait to synchronize tests with actual page loading, eliminating latency failures (flaky tests).

4. Results and Conclusions

I successfully converted a tedious manual checklist into a reliable, automated test suite. I learned to handle complex dynamic elements and delivered clean code with a technical README, ready for integration into a professional workflow.

TEST SUITE 3: Urban Grocers – API Automation

Category: Backend Testing / API

1. The Challenge

The goal was to validate the POST /api/v1/kits endpoint. We needed to ensure the backend correctly applied business rules, rejecting invalid data and handling errors securely. It was suspected the system accepted values that should have been blocked.

2. My Role and Tools

  • Role: API Tester / Automation.
  • Tools: Python, Pytest, Requests Library, Terminal.

3. The Process

I performed a "grey area" analysis of the documentation to find vulnerabilities.

  • Automation: I created scripts that generated dynamic user and authentication tokens for each test.
  • Data Stress Testing: I sent requests with missing data, incorrect data types, and extreme lengths to observe server reaction.
  • Structure: I organized the project modularly (Helpers, Data, Tests) for easy readability.

4. Results and Conclusions

I confirmed that the backend accepted values outside the allowed limits, posing a data integrity risk. I delivered a framework that not only tests if the system "works" but how robust it is against incorrect data.
